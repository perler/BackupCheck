# BackupCheck

A PowerShell-based backup monitoring solution for Macrium Reflect that reports status to healthchecks.io.

## Overview

BackupCheck monitors Macrium Reflect backup repositories by scanning for recent `.mrimg` backup files. It automatically reports the health status of each machine's backups to healthchecks.io, enabling alerting when backups fail or become stale.

**Version:** 0.1.0

## Features

- Monitors multiple backup repositories (local or network shares)
- Auto-detects Macrium Reflect repositories via Site Manager
- Reports to healthchecks.io with auto-provisioning (checks created automatically)
- Skips machines with active backups in progress
- Runs as a Windows Scheduled Task (hourly by default)
- Supports network shares with credential-based access

## Prerequisites

- Windows with PowerShell 5.1 or later
- Macrium Reflect Site Manager installed (for auto-detection of repositories)
- Network access to backup repositories
- healthchecks.io account with a project and Ping Key

## Installation

### 1. Set up healthchecks.io

1. Create an account at [healthchecks.io](https://healthchecks.io)
2. Create a new project for your backups
3. Go to **Project Settings** > **Ping Key** and copy your ping key
4. Configure your alerting integrations (email, Slack, etc.)

### 2. Run the Installer

Open PowerShell as Administrator and run:

```powershell
.\Install-BackupMonitor.ps1
```

The installer will prompt for:
- **Company ID**: A short identifier used as prefix for check names (e.g., `ACME`)
- **Ping Key**: Your healthchecks.io ping key
- **Username/Password**: Credentials for a user that can access backup repositories

The installer will:
1. Auto-detect Macrium Reflect repositories
2. Validate that the provided credentials can access the repositories
3. Create the configuration files
4. Set up a Windows Scheduled Task to run hourly
5. Perform a test run

### 3. Verify Installation

1. Check the Task Scheduler for the `BackupMonitor` task
2. Visit your healthchecks.io dashboard to see the created checks
3. Review the console output from the test run

## Configuration

### config.json

Generated by the installer. Contains:

| Setting | Description | Default |
|---------|-------------|---------|
| `companyId` | Prefix for check names | (set during install) |
| `repositories` | List of backup repository paths | (auto-detected) |
| `backupMaxAgeHours` | Maximum age of backup files in hours | `24` |
| `backupFilePattern` | File pattern for backup files | `*.mrimg` |
| `skipIfRunning` | Skip machines with active backups | `true` |
| `runningFilePattern` | Pattern for "backup running" files | `backup_running*` |
| `healthchecksBaseUrl` | healthchecks.io ping URL | `https://hc-ping.com` |
| `autoDetectRepositories` | Auto-detect repos on each run | `true` |

### .env

Contains the healthchecks.io ping key:

```
HC_PING_KEY=your-ping-key-here
```

## Usage

### Manual Run

```powershell
.\Monitor-Backups.ps1
```

### Check Scheduled Task Status

```powershell
Get-ScheduledTask -TaskName "BackupMonitor"
```

### View Task History

```powershell
Get-ScheduledTask -TaskName "BackupMonitor" | Get-ScheduledTaskInfo
```

## How It Works

1. The script loads configuration from `config.json` and `.env`
2. For each repository, it scans subdirectories (each representing a machine)
3. For each machine:
   - Skips if a `backup_running*` file exists
   - Checks for `.mrimg` files modified within the last 24 hours
   - Sends success or failure ping to healthchecks.io
4. healthchecks.io auto-creates checks on first ping using the `?create=1` parameter

### Check Naming

Checks are named using the pattern: `{companyId}-{machineName}`

Examples:
- `acme-wks001`
- `acme-srv003`

## Troubleshooting

### Checks not appearing in healthchecks.io

1. Verify your ping key is correct in `.env`
2. Run `Monitor-Backups.ps1` manually and check for errors
3. Ensure network connectivity to healthchecks.io

### Access denied to repositories

1. Verify the scheduled task user has read access to all repository paths
2. Re-run the installer with correct credentials
3. Check Windows Credential Manager for stored credentials

### False "down" alerts

1. Check if `backupMaxAgeHours` is appropriate for your backup schedule
2. Verify backup files have the correct `.mrimg` extension
3. Check if backups are being written to unexpected subdirectories

### Scheduled task not running

1. Open Task Scheduler and check the task status
2. Verify "Run whether user is logged on or not" is enabled
3. Check the task history for errors

## Uninstallation

```powershell
# Remove scheduled task
Unregister-ScheduledTask -TaskName "BackupMonitor" -Confirm:$false

# Remove configuration files
Remove-Item config.json, .env
```
